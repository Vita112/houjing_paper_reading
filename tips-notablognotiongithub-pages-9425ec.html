【Tips】 使用Notablog部署，自动同步Notion文章到GitHub Pages

1. how to build it?

  • 在git fork ‣ 仓库后，修改仓库名，用于之后创建网站名，比如修改为houjing_paper_reading；先删掉workflow下的nodejs.yml文件，然后再修改config.json文件中的ulr为自己的notion database的网页公开地址。
  • 新建workflow之前，需要在actions→secrets→actions→New repository secrect创建好token
  • 依次点击仓库下的Actions ->New workflow ->set up a workflow yourself, 会生成 main.yml 文件，将以下代码copy进去，修改部分参考注释
# This is a basic workflow to help you get started with Actions
    
    name: Github Pages
    
    # Controls when the workflow will run
    on:
      schedule:
    # 下一行是说在每天的国际标准时间23点（北京时间早上7点）触发该任务,可以随意修改
        - cron: '0 23 * * *'    
      # Allows you to run this workflow manually from the Actions tab
      workflow_dispatch:
    
    # A workflow run is made up of one or more jobs that can run sequentially or in parallel
    jobs:
      # This workflow contains a single job called "build"
      build:
        # The type of runner that the job will run on
        runs-on: ubuntu-latest
    
        # Steps represent a sequence of tasks that will be executed as part of the job
        steps:
          # 拉取代码
          - name: Checkout
            uses: actions/checkout@v2
          # 1、生成静态文件
          - name: Build
            run: npm i -g notablog && notablog generate . && ls -ilsh ./
          # 2、部署到 GitHub Pages
          - name: Deploy
            uses: JamesIves/github-pages-deploy-action@v4.2.5
            with:
              token: ${{ secrets.ACCESS_TOKEN }}
              repository-name: Vita112/houjing_paper_reading # 修改这里为刚才forked的repo
              branch: doc  # 将创建一个名为doc的repo分支，用于存储部署GitHub Pages的代码                  
              folder: public

  • 关于repository-name，如果你的git地址为git@github.com:Vita112/houjing_paper_reading.git,则为Vita112/houjing_paper_reading ，修改后保存main.yml文件
  • 点击Actions，选择刚才创建的workflow，即 Github Pages ，点击 run workflow，选择master 分支运行，先完成Github Pages后， 自动部署 pages-build-deployment
  • 等这两个 workflow运行结束，前往自己的github pages查看内容。

2. 更新notion，手动部署

前往notion database，新增一条entry，点击 Actions ，选中Github Pages 后，运行run workflow。 等待部署完成，前往网站查看更新。

3. Tips

  • 如果删除notion databese中的某些properties，将导致pages-build-deployment 的自动部署失败,比如
    • date, tags, url, etc.
  • 部署失败后，如何找到失败原因
    • 检查workflow内部运行log，会发现错误，比如build→Build中提示E/notablog-cli(20231101T231342.100): Error: Required property "date" is missing in the Notion table.

4. Customize

  1. Play with templates and stylesheets in notablog-starter/themes/pure/ folder to customize your blog.
